## v2.2-stable

- Included a canonical url on all pages to further boost the SEO experience. [#30](https://github.com/elegantframework/elegant-cli/issues/30)
- Added Storybook JS with dark mode support to Elegant to allow developers to build components in isolation. [#26](https://github.com/elegantframework/elegant-cli/issues/24)
- Added complete sitemaps with robots.txt. [#2](https://github.com/elegantframework/elegant-cli/issues/2)
- Included an `.env.test` environment file to provide Jest variables during unit testing. [#23](https://github.com/elegantframework/elegant-cli/issues/23)
- Introduced enhanced SEO items to the Elegant project, such as rich snippets.
- Created a core components namespace for simplified theming and extendability in future versions of Elegant.
- Improved favicons across all browsers.   
- Slimmed down the existing `.env` file during project creation to only include the basic settings a user might need to start a new app. [#37](https://github.com/elegantframework/elegant-cli/pull/37)
- Included an `.env.example` file that includes all of the essential configuration items a content creator may want. [#37](https://github.com/elegantframework/elegant-cli/pull/37)
- Improved the output message displayed to users during project creation. [#35](https://github.com/elegantframework/elegant-cli/pull/35)
- Included an auto generated Trademark policy page that includes basic business details loaded from the config file. [#44](https://github.com/elegantframework/elegant-cli/pull/44)


<a name="v2_1-beta"></a>

## v2.1-beta

- Added native Google Analytics support via the .env file. [Google Analytics configuration](/docs/google-analytics). [#1](https://github.com/elegantframework/elegant-cli/issues/1)
- Added native Tailwind theme colors inside tailwind.config.js file. [Theme configuration](/docs/theme)
- Added native ConvertKit newsletter sign up support via the .env file. [ConvertKit configuration](/docs/convertkit)
- Updated the Elegant CLI init script to provide detailed instructions for the build steps when starting a new project. [#13](https://github.com/elegantframework/elegant-cli/issues/13)
- Created a new core `Splash Page` layout. The splash page layout can be used for creating landing pages with sticky headers and custom navigation. [#16](https://github.com/elegantframework/elegant-cli/commit/d8f657af269bda072615b42dca76184b384f4188)
- Added a sticky header meta page option. 
- Created `core` components namespace. The core components will function as a starting point for future theme creation and extendability. [#17](https://github.com/elegantframework/elegant-cli/pull/17)
- Created a new "Splash" landing page layout template for creating simple landing pages.
- Updated the footer url links to pop open in a new tab if they are linking to external urls.

## v2.0-alpha

- Added Jest testing to the library.

- Added Elegant base theme with a near perfect Lighthouse SEO score.
